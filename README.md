A console-based Bank Management System developed in C++ that models real-world banking functionality while demonstrating the practical use of core data structures and object-oriented design. Customer accounts are maintained using a singly linked list, where each node represents a customer containing a unique auto-generated ID, name, balance, and associated transaction data. Every financial operation—such as deposits, withdrawals, and transfers—is recorded in a transaction queue (implemented via a linked list) that stores the transaction amount, reason, and a timestamp generated at runtime.

The system provides robust transaction management by incorporating an undo/redo mechanism. When a transaction is undone, it is removed from the transaction queue and pushed onto a stack, allowing it to be restored later through a redo operation. This effectively demonstrates the use of stacks for reversing operations. Transfers between accounts are handled carefully by performing both withdrawal and deposit steps with appropriate validation, while restricting undo functionality for transfer operations to maintain consistency.

Additional features include input validation for numeric values, minimum balance enforcement during account creation, and prevention of invalid operations such as overdrawing funds or transferring to the same account. The program also tracks and displays execution time (in nanoseconds) for major operations using the `<chrono>` library, giving insight into performance.

Overall, this project showcases the integration of linked lists, queues, and stacks in a cohesive system, along with concepts such as dynamic memory allocation, pointer manipulation, modular class design, and real-time data handling in a banking simulation environment.
